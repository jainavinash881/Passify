## Permissions Explanation

Passify requests the following browser permissions:

### 1. **Storage Permission**
- **Purpose:** Store your passwords and settings locally on your device
- **Scope:** Local storage only, no cloud sync
- **Data:** Passwords, usernames, website URLs, preferences

### 2. **Active Tab Permission**
- **Purpose:** Detect password fields on the current webpage
- **Scope:** Only the active tab when you interact with the extension
- **Data:** No data is collected, only used to inject autofill functionality

### 3. **Tabs Permission**
- **Purpose:** Open the password manager page in a new tab
- **Scope:** Only to create new tabs when you click "Manage Passwords"
- **Data:** No tab data is collected or stored

### 4. **Host Permissions (All URLs)**
- **Purpose:** Inject autofill functionality on any website
- **Scope:** Only activates on pages with password fields
- **Data:** No data is collected from websites

**All permissions are used solely for the extension's core functionality and nothing else.**

## Data Security

### How Your Data Is Protected:

1. **Local Storage:** All data is stored using browser's secure storage APIs
2. **No Transmission:** Data never leaves your device
3. **No Encryption Needed:** Since data never leaves your device, there's no transmission to encrypt
4. **Browser Security:** Your data is protected by your browser's built-in security features
5. **No External Access:** We cannot access your data even if we wanted to

### Your Responsibility:
- Keep your device secure
- Use a strong device password
- Keep your browser updated
- Use browser's built-in security features
